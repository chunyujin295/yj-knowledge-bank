# 什么是大模型

## 概述

「大模型」是**大语言模型（Large Language Model, LLM）** 的简称——用海量文本数据训练出来的、参数量极大的神经网络模型。
它们最核心的能力是**理解和生成自然语言**，但模型大到一定程度后还展现出了推理、编程、翻译等「涌现能力」。
本文从零开始解释大模型到底是什么、为什么「大」是关键、以及它们是怎么发展起来的。

> [!note] 前置知识
> 本文不需要任何机器学习背景。如果你已经了解基本概念，可以直接跳转到 [[大模型技术原理]] 深入了解架构细节。

## 1. 什么是语言模型？

### 1.1 最简定义

**语言模型 = 预测下一个词的概率分布。**

给一个句子 "我今天早上喝了一杯 ___"，语言模型会给出每个可能的词的概率：

```
P(咖啡 | 我今天早上喝了一杯) = 0.35
P(牛奶 | 我今天早上喝了一杯) = 0.28
P(茶   | 我今天早上喝了一杯) = 0.20
P(啤酒 | 我今天早上喝了一杯) = 0.01
...
```

这就是语言模型要做的最基本的事：**给定上文，预测下文**。

### 1.2 从 N-gram 到神经网络

```
传统 N-gram 模型:
  P(喝|我,今天,早上) ≈ 统计语料中 "我今天早上喝" 的出现次数
  问题: 词组合爆炸 — "我今天早上喝咖啡然后去..." 在语料中可能从未出现

神经网络语言模型:
  将每个词映射为向量 → 通过神经网络学习词之间的关系 → 泛化到未见过的组合
  关键: "咖啡" 和 "茶" 的向量很近，所以 "喝咖啡" 和 "喝茶" 都能理解
```

## 2. 大模型的「大」是什么？

### 2.1 参数量的概念

模型的核心结构是**神经网络**，网络中的每个连接权重就是一个**参数**。

```
一个小型的传统 NLP 模型:
  Word2Vec:       ~1M  参数    (100万)
  LSTM 分类器:     ~10M 参数    (1000万)

大模型:
  GPT-2:          ~1.5B 参数    (15亿)
  GPT-3:          ~175B 参数    (1750亿)
  GPT-4:          未公开 (估计 >1T)  (万亿级)
  Llama 3 405B:   ~405B 参数
```

> [!info] 直观类比
> 如果把每个参数印在一张 A4 纸上，GPT-3 的 1750 亿参数需要约 1750 亿张纸——足够堆满好几个仓库。

### 2.2 训练数据有多大？

不仅仅是参数多，训练数据也是海量的：

| 模型 | 训练数据量 | 相当于 |
|------|-----------|--------|
| GPT-3 | ~300B tokens | 约 45 万本书（按 70 万 tokens/书） |
| Llama 2 | ~2T tokens | 约 300 万本书 |
| Llama 3 | ~15T tokens | 约 2200 万本书 |

一个普通人一生大约阅读 1000 本书。大模型「读过」的书相当于几十万人的一生阅读量。

## 3. 大模型发展简史

```
2017  "Attention Is All You Need" 论文发布 → Transformer 架构诞生
      │
2018  GPT-1 (117M), BERT (340M) → 证明了预训练 + 微调范式的有效性
      │
2019  GPT-2 (1.5B) → 展示了零样本能力，OpenAI 一度不敢开源
      │
2020  GPT-3 (175B) → 涌现能力首次被系统观察，Few-shot 学习惊艳
      │
2022  ChatGPT (GPT-3.5) → 加入 RLHF，会对话了，全球爆火
      │
2023  GPT-4, Llama 2, Claude → 多模态、开源、安全对齐成为主题
      │
2024  Llama 3, GPT-4o, Claude 3.5, DeepSeek-V3 → 推理能力、多模态融合、MoE 架构
      │
2025  DeepSeek-R1, GPT-5, Claude 4 → 推理时代：模型学会「思考」
```

## 4. 缩放法则（Scaling Law）—— 为什么越大越好？

### 4.1 核心发现

OpenAI 在 2020 年发表的论文发现：**模型性能与参数量、数据量、计算量之间存在可预测的幂律关系**。

```
损失(Loss) ∝ (参数量)^(-α) × (数据量)^(-β) × (计算量)^(-γ)

简化为:
  参数越多 → 损失越低（性能越好）
  数据越多 → 损失越低
  算力越多 → 损失越低

没有明显瓶颈，只要三个量同时增长，性能就能稳定提升。
```

### 4.2 涌现能力（Emergence）

最令人惊讶的发现是：**有些能力不是慢慢变好的，而是突然出现的**。

```
能力强度
    │                                    ┌─────── 推理
    │                              ┌─────┤
    │                        ┌─────┤      └─────── 翻译
    │                  ┌─────┤
    │            ┌─────┤      └────────── 代码生成
    │      ┌─────┤
    │──────┤      └──────────────────── 文本补全 (线性增长)
    │
    └──────┴──────┴──────┴──────┴──────► 模型规模 (参数数量)
         10⁸    10⁹    10¹⁰   10¹¹   10¹²

在约 10¹⁰ (10B) 参数时，很多高级能力「突然」涌现出来。
模型在 6B 时几乎不会做算术，到 13B 时突然能解数学题了。
```

常见的涌现能力包括：
- **数学推理**：多步算术、方程求解
- **代码生成**：根据自然语言描述写代码
- **翻译**：在未专门训练过的语言对之间翻译
- **思维链 (Chain-of-Thought)**：分步骤推理
- **上下文学习 (In-Context Learning)**：通过几个示例就学会新任务

## 5. Token —— 模型眼中的「文字」

### 5.1 Token 是什么？

模型并不是逐字阅读文本的。文本首先被切分为 **Token（词元）**：

```
输入: "大模型是一种人工智能技术"

Tokenization:
  大 | 模型 | 是 | 一种 | 人工智能 | 技术
  (某些 tokenizer 可能会进一步拆分: 人工智能 → 人工 | 智能)

英文示例:
  "unbelievable" → ["un", "believe", "able"]
```

### 5.2 为什么不直接用字？

| 方式 | 优点 | 缺点 |
|------|------|------|
| 逐字 (Character) | 词汇表小 | 序列太长，语义分散 |
| 分词 (Word) | 语义完整 | 词汇表巨大，OOV 问题 |
| **子词 (Subword/Token)** | 平衡两者 | 对中文有时切分不自然 |

> [!info] 中文的特殊性
> 中文的 tokenization 比英文复杂。常见做法是先分词再子词化。
> 1 个中文字 ≈ 1.5~2 个 tokens（不同 tokenizer 不同）。
> GPT 系列的中文效率偏低（一个汉字可能 2-3 tokens），而国产模型如 DeepSeek、Qwen 对此做了优化。

## 6. 大模型的训练范式

大模型的完整训练通常分为几个阶段：

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   预训练       │ ──► │   监督微调    │ ──► │   对齐训练    │
│   Pretraining │     │     SFT      │     │   RLHF/DPO   │
├──────────────┤     ├──────────────┤     ├──────────────┤
│ 数据: 海量网页  │     │ 数据: 高质量QA  │     │ 数据: 人类偏好  │
│ 目标: 预测下一词│     │ 目标: 学会对话  │     │ 目标: 对齐人类  │
│ 代价: 极高     │     │ 代价: 低       │     │ 代价: 中等     │
└──────────────┘     └──────────────┘     └──────────────┘
```

## 8. 常见误区澄清

| 误区 | 事实 |
|------|------|
| "大模型真的理解语言" | 大模型是统计模型，通过概率分布生成文本，是否「理解」是哲学问题 |
| "参数越多越聪明" | 小模型 + 高质量数据可以超过大模型 + 差数据（见 Chinchilla 定律） |
| "大模型会思考" | 它们做的是模式匹配和概率推理，不是人类意义上的思考 |
| "大模型能记住训练数据" | 大模型是压缩而非存储，不能逐字还原训练数据（但可能近似复现） |
| "越大越好" | 存在收益递减，且推理成本随规模快速增长 |
