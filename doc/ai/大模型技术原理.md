# 大模型技术原理

## 概述

所有现代大模型都基于同一个核心架构：**Transformer**。本文从零推导 Transformer 的设计思路，逐层拆解 Self-Attention、Multi-Head Attention、位置编码和前馈网络，最后给出一个完整的推理流程走读。

> [!note] 前置知识
> 建议先阅读 [[什么是大模型]]，了解 Token、参数等基础概念。

## 1. 为什么需要 Transformer？

### 1.1 序列建模的挑战

要理解一段文字，模型必须解决一个核心问题：**词与词之间的关系可能跨越很长的距离**。

```
"我昨天在街角那家开了二十多年的老咖啡馆里点了一杯拿铁，味道真的很（好）"

要预测"好"，模型需要知道：
  - "味道" 是主语 → 距离 6 个词
  - "拿铁" 是修饰对象 → 距离 10 个词
  - "咖啡馆" 是背景 → 距离 15 个词
```

### 1.2 RNN 的问题

在 Transformer 之前，主流方案是 RNN（循环神经网络）：

```
RNN 处理方式（串行）:
  词1 → [RNN] → 状态1
                词2 → [RNN] → 状态2
                              词3 → [RNN] → 状态3
                                              ...

  问题:
  ┌─────────────────────────────────────────────────┐
  │ 1. 串行计算：不能并行，训练极慢                      │
  │ 2. 长距离依赖：经过 20 个词后，前面的信息几乎丢失       │
  │ 3. 梯度消失/爆炸：反向传播时梯度越来越小/大            │
  └─────────────────────────────────────────────────┘
```

**Transformer 的核心创新 = 并行处理整个序列 + Attention 机制直接建模任意距离的依赖。**

## 2. Transformer 总览

```
                    ┌───────────────────┐
                    │      Output       │
                    │   (下一个 Token)   │
                    └────────┬──────────┘
                             │
                    ┌────────┴──────────┐
                    │  Linear + Softmax │
                    └────────┬──────────┘
                             │
              ┌──────────────┴──────────────┐
              │     ↑ 残差连接 + LayerNorm   │
              │  ┌───────────────────────┐  │
              │  │   Feed-Forward Net    │  │
              │  │   (两层全连接网络)      │  │
              │  └───────────────────────┘  │
              │     ↑ 残差连接 + LayerNorm   │
              │  ┌───────────────────────┐  │
              │  │  Multi-Head Attention │  │
              │  │     (核心机制)         │  │
              │  └───────────────────────┘  │
              │                             │
              │     × N (重复 N 层)          │
              └─────────────────────────────┘
                             ↑
              ┌──────────────┴──────────────┐
              │  Input Embedding + 位置编码   │
              └──────────────┬──────────────┘
                             ↑
                    ┌────────┴──────────┐
                    │   Input Tokens    │
                    └───────────────────┘

  这是 Decoder-Only 架构（GPT 系列使用）
  Encoder-Decoder 架构（如原始 Transformer、T5）多了左边的 Encoder 部分
```

> [!info] Decoder-Only vs Encoder-Decoder
> - **Encoder-Decoder**（BERT, T5）：Encoder 编码输入，Decoder 生成输出。适合翻译等需要完整理解输入的任务。
> - **Decoder-Only**（GPT, Llama）：只用 Decoder，适合文本生成。是目前大模型的主流架构。
> - **Encoder-Only**（BERT）：只编码，适合分类、提取等理解任务。

## 3. Self-Attention — 核心机制

### 3.1 直觉

Self-Attention 回答的问题是：**对于序列中的每个词，其他词和它有多相关？**

```
输入句子: "猫 吃 鱼 因为 它 饿 了"

对于 "它"，哪些词最相关？
  猫:  ████████████████████░░░░  0.7  (它很可能指"猫")
  吃:  ██░░░░░░░░░░░░░░░░░░░░  0.1
  鱼:  █░░░░░░░░░░░░░░░░░░░░░░  0.05
  饿:  ████████████░░░░░░░░░░░  0.4  (饿和它也相关)
  ...

每个词都和其他所有词计算一个注意力分数，然后用这个分数加权融合信息。
```

### 3.2 Q、K、V — 三大核心向量

Self-Attention 用三个矩阵将每个 Token 映射为三个向量：

```
对于每个 Token 的向量 x:

  Q = x × W_Q     Query  (查询)  — "我在找什么？"
  K = x × W_K     Key    (键)    — "我是什么？"
  V = x × W_V     Value  (值)    — "我包含什么信息？"

  ┌──────────────────────────────────────────────┐
  │  Q 和 K 用来计算「相关性分数」（注意力权重）       │
  │  V 用这个分数加权求和，得到最终的「关注结果」       │
  └──────────────────────────────────────────────┘
```

> [!info] 类比：图书馆检索
> - **Q (Query)**：你在图书馆系统里输入的搜索词
> - **K (Key)**：每本书的标题和标签
> - **V (Value)**：每本书的实际内容
> - 系统将你的 Query 和所有书的 Key 匹配，然后按匹配度加权返回 Value（内容）

### 3.3 计算步骤（逐步走读）

设输入序列长度为 `n`，每个 Token 的向量维度为 `d`（如 4096）：

```
步骤 1: 生成 Q, K, V

  X = [x₁, x₂, ..., xₙ]    shape: (n, d)

  Q = X @ W_Q              shape: (n, d_k)    d_k = d / num_heads
  K = X @ W_K              shape: (n, d_k)
  V = X @ W_V              shape: (n, d_k)

步骤 2: 计算注意力分数

  Scores = Q @ Kᵀ          shape: (n, n)

  每一行对应一个 Token 对所有 Token 的原始相关度分数

步骤 3: 缩放（防止梯度消失）

  Scores = Scores / √d_k

  为什么除以 √d_k？
  - 当 d_k 很大时，点积结果的值域很大
  - 过大的值输入 Softmax 会导致梯度趋近于 0
  - 除以 √d_k 使方差稳定在 1

步骤 4: Softmax（转为概率分布）

  Attention Weights = Softmax(Scores)   shape: (n, n)

  每行的值在 [0, 1] 之间，且和为 1

步骤 5: 加权求和

  Output = Attention Weights @ V    shape: (n, d_k)

  每个 Token 的输出 = 所有 Token 的 V 的加权和，权重是注意力分数
```

### 3.4 因果掩码（Causal Mask）

在文本生成中，模型不应该「看到未来」。因此在 Decoder 中使用因果掩码：

```
注意力矩阵 (n=4):

  "我"  "喝"  "了"  "咖啡"
┌─────────────────────────┐
│ 1.0   0     0     0   │  "我"   只能看到自己
│ 0.3  0.7    0     0   │  "喝"   只能看到 "我" "喝"
│ 0.2  0.3   0.5    0   │  "了"   只能看到前三个
│ 0.1  0.2   0.2   0.5  │  "咖啡" 可以看到所有
└─────────────────────────┘

实现: 将上三角区域设为 -∞，Softmax 后自动变为 0
```

## 4. Multi-Head Attention（多头注意力）

### 4.1 为什么需要多头？

单个 Attention 只能捕捉一种类型的关系。多头注意力让模型同时关注多种不同的关系：

```
单头:
  "她 把 书 递给 他"
  可能只关注: "她" ↔ "书"

多头:
  Head 1: 主语-谓语关系    "她" → "递给"
  Head 2: 宾语关系        "书" → "递给"
  Head 3: 指代关系        "他" → 接收者
  Head 4: 词法关系        "书" → "把" 结构
  ...
```

### 4.2 计算方式

```
将 d_model 均分为 h 组，每组独立计算 Attention

  d_model = 4096, h = 32 (Llama 3 8B)
  每个 Head 处理: d_head = 4096 / 32 = 128 维

  ┌─────────────────────────────────────────────┐
  │ Head₁: Q₁ = XW_Q₁, K₁ = XW_K₁, V₁ = XW_V₁  │
  │        → Attention₁ = Softmax(Q₁K₁ᵀ/√dₖ)V₁  │
  │                                             │
  │ Head₂: Q₂ = XW_Q₂, K₂ = XW_K₂, V₂ = XW_V₂  │
  │        → Attention₂ = Softmax(Q₂K₂ᵀ/√dₖ)V₂  │
  │                                             │
  │ ...   (并行计算，GPU 友好)                    │
  │                                             │
  │ Head₃₂: ...                                 │
  └─────────────────────────────────────────────┘
                          │
                          ▼
  Concat + Linear 投影:
  Output = Concat(Head₁, ..., Head₃₂) @ W_O    shape: (n, d_model)
```

## 5. Feed-Forward Network（前馈网络）

每个 Attention 层后面跟着一个 FFN：

```
FFN(x) = ReLU(x @ W₁ + b₁) @ W₂ + b₂

或使用 SwiGLU (Llama 等现代模型):
FFN(x) = (Swish(x @ W_gate) ⊙ (x @ W_up)) @ W_down

                                输入 (d_model)

                          ┌───────┴───────┐
                          │               │
                      Gate 投影        Up 投影
                      (d_ff)          (d_ff)
                          │               │
                      Swish激活            │
                          │               │
                          └───┬───────────┘
                              │
                         逐元素相乘
                              │
                          Down 投影 ───────► 输出 (d_model)

d_ff 通常是 d_model 的 2.5~4 倍（Llama 3 8B: d_ff = 14336）
```

> [!info] FFN 的作用
> Attention 负责「信息路由」——决定从哪些位置获取信息。
> FFN 负责「信息变换」——对汇总的信息进行非线性处理，存储知识。
> 有研究表明，FFN 层是模型「知识存储」的主要位置。

## 6. 位置编码

Attention 本身不包含位置信息——「猫吃鱼」和「鱼吃猫」的 Attention 分数可能一模一样。

### 6.1 绝对位置编码

```
原始 Transformer 使用正弦函数:

PE(pos, 2i)   = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

  位置 0: [sin(0), cos(0), sin(0/100), cos(0/100), ...]
  位置 1: [sin(1), cos(1), sin(1/100), cos(1/100), ...]

  特点: 固定不变（非学习参数），可以外推到训练时未见过的长度
```

### 6.2 旋转位置编码（RoPE）

**RoPE (Rotary Position Embedding)** 是目前主流方案（Llama、Qwen、DeepSeek 都用它）：

```
核心思想: 通过旋转操作将位置信息融入 Q 和 K

  Q_rot = RoPE(Q, pos)
  K_rot = RoPE(K, pos)

  旋转角度 = pos × θ_i，其中 θ_i 是频率参数

  这样做的好处:
  ┌──────────────────────────────────────────┐
  │ 1. 只有相对位置影响注意力，而非绝对位置     │
  │ 2. 自然衰减：距离越远，注意力分数越低       │
  │ 3. 易于外推：可以推广到训练时未见过的长度    │
  └──────────────────────────────────────────┘
```

## 7. 残差连接 + Layer Normalization

每个子层（Attention 和 FFN）都包裹在残差连接和层归一化中：

```
Pre-Norm (GPT-3, Llama 等现代模型使用):

  输出 = x + SubLayer(LayerNorm(x))

  ┌──────┐    ┌──────┐    ┌──────┐    ┌───────┐
  │ 输入  │───►│Norm  │───►│Attn  │───►│ + 残差 │───►
  └──────┘    └──────┘    └──────┘    └──┬┬───┘
                                         ││
                  ┌───────────────────────┘│
                  │  残差: 将原始输入直接加回来
                  └────────────────────────┘

  Pre-Norm vs Post-Norm:
  - Post-Norm (原始 Transformer): 在残差之后做 Norm → 训练不稳定
  - Pre-Norm (现代做法): 在子层之前做 Norm → 训练更稳定，无需 warmup
```

## 8. 完整推理走读

以下是一次推理的完整数据流动：

```
输入: "我 今天 心情"

┌──────────────────────────────────────────────────────────┐
│ Step 1: Token Embedding                                   │
│                                                          │
│   "我"    → [0.12, -0.34, ..., 0.56]  (4096维向量)       │
│   "今天"  → [0.78, 0.21, ..., -0.11]                     │
│   "心情"  → [-0.45, 0.67, ..., 0.33]                     │
│                                                          │
│   查表操作：vocab_size × d_model 的 Embedding Matrix      │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ Step 2: 加位置编码 (RoPE)                                  │
│                                                          │
│   x₁ = embed("我")   + rope(0)                           │
│   x₂ = embed("今天") + rope(1)                           │
│   x₃ = embed("心情") + rope(2)                           │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ Step 3: 通过 N 层 Transformer Block                       │
│                                                          │
│   For each layer:                                        │
│     x = x + MultiHeadAttention(LayerNorm(x))             │
│     x = x + FFN(LayerNorm(x))                            │
│                                                          │
│   经过 32 层 (Llama 3 8B) → 输出最后位置的 hidden state     │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ Step 4: LM Head (输出层)                                   │
│                                                          │
│   logits = hidden_state @ W_lm  (W_lm 和 Embedding 共享)  │
│   logits shape: (1, vocab_size) — 每个词的概率分数         │
│                                                          │
│   例如: vocab_size = 128,000 个 tokens                    │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ Step 5: 采样                                              │
│                                                          │
│   probs = Softmax(logits / temperature)  ← 温度控制随机性  │
│   next_token = Sample(probs) 或 Argmax(probs)            │
│                                                          │
│   结果: "好" (概率最高) → 拼接到输入 → 重复 Step 1-5     │
└──────────────────────────────────────────────────────────┘
```

## 9. 主流架构变体

### 9.1 MoE (Mixture of Experts)

```
传统 Dense 模型:
  每个 Token 激活全部参数

MoE 模型 (如 Mixtral, DeepSeek-V3):
  ┌──────────┐
  │  Router   │  ← 路由器决定每个 Token 送往哪几个 Expert
  └────┬─────┘
       │
  ┌────┼────┬────┬────┐
  ▼    ▼    ▼    ▼    ▼
┌────┐┌────┐┌────┐┌────┐┌────┐
│Expt││Expt││Expt││Expt││Expt│ ... (共 N 个 Expert)
│ 1  ││ 2  ││ 3  ││ 4  ││ 5  │
└────┘└────┘└────┘└────┘└────┘
  每个 Token 只激活 Top-K 个 Expert (通常 K=2~8)

  优点:
  - 总参数巨大 (DeepSeek-V3: 671B)，但每个 Token 只激活部分 (37B)
  - 推理更快，训练更省
```

### 9.2 GQA (Grouped Query Attention)

```
MHA (Multi-Head Attention):
  Q头 = K头 = V头  (如 32 个)

MQA (Multi-Query Attention):
  所有 Q 头共享 1 组 K 和 V  (KV: 1 组)

GQA (Grouped-Query Attention):
  Q 头分成 G 组，每组共享 1 组 K 和 V  (KV: 4~8 组)

  ┌──────────────────────────────────┐
  │ Q₁ Q₂ ... Q₃₂                   │  ← 32 个 Query Head
  │  ↓   ↓       ↓                   │
  │  K₁  K₂ ... K₈                  │  ← 8 个 KV Head
  └──────────────────────────────────┘

  这是目前主流做法 (Llama 3, Qwen 2.5 都用 GQA)
  好处: 大幅减少 KV Cache 内存占用，推理更快
```

## 关键概念速查表

| 中文概念 | 英文 | 一句话解释 |
|----------|------|-----------|
| Transformer | Transformer | 基于自注意力的神经网络架构，现代大模型的基石 |
| 自注意力 | Self-Attention | 让每个位置关注所有其他位置并加权融合信息的机制 |
| 查询/键/值 | Query/Key/Value | 注意力计算的三个核心向量：Q 查询、K 匹配、V 提供信息 |
| 多头注意力 | Multi-Head Attention | 并行运行多组注意力，从不同角度捕获语言关系 |
| 前馈网络 | Feed-Forward Network | Attention 后的全连接层，负责信息变换和知识存储 |
| 位置编码 | Positional Encoding | 为模型提供 Token 在序列中位置信息的方法 |
| 旋转位置编码 | RoPE | 通过对 Q 和 K 向量旋转来编码相对位置信息 |
| 残差连接 | Residual Connection | 将输入直接加到输出上，解决深层网络的训练困难 |
| 层归一化 | Layer Normalization | 对每层的激活值做归一化，稳定训练 |
| 因果掩码 | Causal Mask | 防止 Token 关注未来位置，保证自回归生成的因果性 |
| 混合专家 | MoE | 多个「专家」子网络 + 路由器，稀疏激活提高效率 |
| 分组查询注意力 | GQA | 多个 Q Head 共享 KV Head，减少内存和计算 |
