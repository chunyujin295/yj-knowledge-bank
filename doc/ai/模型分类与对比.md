# 模型分类与对比

## 概述

「大模型」只是 AI 模型版图中的一块。本文梳理从传统机器学习到大语言模型、视觉模型、多模态模型的完整分类体系，帮助你理解不同模型的能力边界、适用场景和相互关系。

> [!note] 前置知识
> 建议先阅读 [[什么是大模型]] 了解 LLM 的基本概念。

## 2. 按「学习范式」分类

### 2.1 判别式模型（Discriminative）

```
目标: 学习 P(标签|输入) — 给定输入，判断它属于哪个类别

            输入 ──► [模型] ──► 分类/回归结果

典型任务:
  - 分类: 这封邮件是垃圾邮件吗？
  - 回归: 这个房子值多少钱？
  - 检测: 图片中哪里有人脸？

代表模型:
  - 传统 ML: SVM、随机森林、XGBoost、逻辑回归
  - 深度学习: CNN (ResNet)、RNN、BERT (分类模式)
```

### 2.2 生成式模型（Generative）

```
目标: 学习 P(输入) — 学习数据的分布，能够生成新数据

            输入/噪声 ──► [模型] ──► 生成的内容

典型任务:
  - 文本生成: 写一篇文章
  - 图像生成: 画一幅画
  - 代码生成: 写一个函数

代表模型:
  - 文本: GPT、Llama、Claude (自回归)
  - 图像: Stable Diffusion、DALL-E、Midjourney (扩散)
  - 视频: Sora、Runway
```

### 2.3 强化学习模型（Reinforcement Learning）

```
目标: 学习最优策略 π(a|s) — 在什么状态下该做什么动作

        环境 ──► 状态 ──► [Agent] ──► 动作 ──► 奖励 ──► 环境
          ▲                                      │
          └──────────────────────────────────────┘

典型任务:
  - 游戏 AI (AlphaGo, OpenAI Five)
  - 机器人控制
  - 自动驾驶决策
  - LLM 的 RLHF 对齐 (间接使用)
```

## 3. 大语言模型（LLM）细分

### 3.1 按架构分类

| 架构 | 代表模型 | 适用场景 | 当前地位 |
|------|---------|----------|----------|
| **Encoder-Only** | BERT, RoBERTa, DeBERTa | 文本分类、实体识别、抽取式 QA | 小模型使用，大模型不用 |
| **Decoder-Only** | GPT 系列, Llama, Claude, Qwen | 文本生成、对话、推理 | **大模型主流** |
| **Encoder-Decoder** | T5, BART, GLM | 翻译、摘要、需要深度理解的任务 | 少数场景使用 |

```
为什么 Decoder-Only 成为主流？

1. 自回归生成自然匹配语言 — 人说话也是一个一个词说的
2. 架构统一简单 — 不需要区分 Encoder 和 Decoder
3. 上下文学习能力强 — GPT-3 证明了 Few-shot 的威力
4. 训练效率高 — 所有 Token 都可以参与损失计算（next-token prediction）
```

### 3.2 按开源/闭源分类

| 类型 | 代表 | 优点 | 缺点 |
|------|------|------|------|
| **闭源** | GPT-4o, Claude 4, Gemini 2.5 | 性能顶尖，有 API | 不可本地部署，成本高，数据隐私问题 |
| **开源** | Llama 3, Qwen 2.5, DeepSeek-V3 | 本地部署，可微调，数据安全 | 需要自己搭建推理环境 |

### 3.3 按规模分类

| 规模 | 参数量 | 代表模型 | 典型硬件需求 |
|------|--------|----------|-------------|
| **小型** | <3B | Phi-3, Qwen 2.5-1.5B | 手机端可运行 |
| **中型** | 7B-13B | Llama 3-8B, Qwen 2.5-7B | 消费级 GPU (RTX 3090/4090) |
| **大型** | 70B+ | Llama 3-70B, Qwen 2.5-72B | 多卡 (2-4× A100) |
| **超大型** | 400B+ | Llama 3-405B, DeepSeek-V3(671B) | 服务器集群 |

## 4. 视觉模型（Vision Models）

### 4.1 扩散模型（Diffusion Model）

扩散模型是**图像生成**的主流方案，通过逐步去噪生成图像：

```
训练过程:
  清晰图 ──► 逐步加噪 ──► 纯噪声
              (前向过程)
              [模型学习如何反向去噪]

推理过程:
  纯噪声 ──► 逐步去噪 ──► 生成的图像
              (反向过程，通常需要 20-50 步)

代表模型:
  - Stable Diffusion (开源，主流)
  - DALL-E 3 (OpenAI)
  - Midjourney (闭源，图像质量顶尖)
  - Flux (开源，新一代)
```

> [!info] 注意
> 扩散模型专注于**图像生成**。Stable Diffusion 是开源的代表，生态最丰富（LoRA、ControlNet 等社区贡献）。

### 4.2 传统视觉模型

| 类型 | 代表 | 任务 |
|------|------|------|
| CNN | ResNet, EfficientNet | 图像分类、目标检测 |
| ViT | Vision Transformer | 图像分类（Transformer 思路用图） |
| YOLO | YOLOv8/v9/v10 | 实时目标检测 |
| SAM | Segment Anything | 图像分割 |

## 5. 多模态模型（Multimodal Models）

多模态模型能同时处理多种类型的数据（通常是文本 + 图像）。

### 5.1 视觉语言模型（VLM）

```
                    ┌─────────────┐
      图像 ────────►│  Vision      │
                    │  Encoder    │──► 图像特征
                    └─────────────┘       │
                                          ├──► [LLM] ──► 回答
                    ┌─────────────┐       │
      文本 ────────►│  Tokenizer  │──► 文本 Token
                    └─────────────┘

典型架构: ViT (图像编码器) + Projector (对齐层) + LLM (文本模型)

代表模型:
  - GPT-4o / GPT-5 (OpenAI) — 多模态原生
  - Gemini 2.5 (Google) — 多模态原生
  - Claude 4 (Anthropic) — 多模态原生
  - Llama 3.2 Vision (Meta) — 开源
  - Qwen-VL (阿里) — 开源
  - LLaVA — 学术界经典方案 (CLIP ViT + Vicuna)
```

### 5.2 其他多模态方向

| 类型 | 能力 | 代表模型 |
|------|------|----------|
| 文生图 | 文字描述 → 生成图像 | DALL-E, Stable Diffusion, Midjourney |
| 文生视频 | 文字描述 → 生成视频 | Sora, Runway Gen-3, 可灵 |
| 文生音乐 | 文字描述 → 生成音乐 | Suno, Udio |
| 图生文 | 图像 → 文字描述 | BLIP-2, 各种 VLM |
| 语音大模型 | 语音理解与生成 | GPT-4o Voice, Moshi |
| 世界模型 | 视频 → 预测未来帧 | Sora (有争议) |

## 6. 大模型 vs 传统模型 — 核心差异

| 维度 | 传统 ML/DL 模型 | 大语言模型 |
|------|----------------|-----------|
| **训练数据** | 标注数据（昂贵） | 海量无标注数据（便宜） |
| **任务范围** | 单一任务 | 通用任务，通过 prompt 切换 |
| **训练方式** | 从零训练 | 预训练 + 微调 |
| **模型大小** | MB ~ 几百 MB | GB ~ TB |
| **推理能力** | 无 | 涌现出一定推理能力 |
| **幻觉问题** | 不存在 | 可能生成虚假信息 |
| **可解释性** | 相对可解释 | 黑箱，难解释 |
| **部署成本** | 低（CPU 可跑） | 高（需要 GPU） |
| **更新方式** | 重新训练 | RAG、微调、提示工程 |

## 7. 如何选择合适的模型？

```
          你的任务是什么？
                │
    ┌───────────┼───────────┐
    │           │           │
  文本        图像        多模态
    │           │           │
    ▼           ▼           ▼
┌────────┐ ┌────────┐ ┌────────────┐
│需要推理?│ │生成?   │ │需要看图和文字?│
│        │ │        │ │            │
│是→LLM  │ │是→扩散  │ │是→VLM       │
│否→BERT │ │否→CNN/ │ │            │
│        │ │  ViT   │ │            │
└────────┘ └────────┘ └────────────┘

选择 LLM 时:
┌────────────────────────────────────────────────────┐
│ 1. 需要本地部署？ → 开源模型 (Llama, Qwen, DeepSeek)│
│ 2. 追求性能？    → 闭源 API (GPT-4o, Claude 4)      │
│ 3. 中文场景？    → Qwen, DeepSeek, GLM             │
│ 4. 资源有限？    → Phi-3, Qwen 2.5-1.5B, 量化版本   │
│ 5. 特定领域？    → 微调开源模型                      │
└────────────────────────────────────────────────────┘
```

## 关键概念速查表

| 中文概念 | 英文 | 一句话解释 |
|----------|------|-----------|
| 判别式模型 | Discriminative Model | 学习输入到标签的映射，用于分类和回归 |
| 生成式模型 | Generative Model | 学习数据分布，能生成新样本 |
| 扩散模型 | Diffusion Model | 通过逐步去噪生成图像的主流模型 |
| 视觉语言模型 | VLM | 能同时理解图像和文本的多模态模型 |
| 多模态模型 | Multimodal Model | 能处理文本、图像、语音等多种模态的模型 |
| MoE | Mixture of Experts | 多个专家子网络 + 路由器的稀疏激活架构 |
| 自回归 | Autoregressive | 逐个 Token 生成，每个新 Token 依赖之前生成的所有内容 |
| 编码器-解码器 | Encoder-Decoder | 先编码理解输入，再解码生成输出的架构 |
| 端侧推理 | On-Device Inference | 在手机、PC 等终端设备上本地运行模型 |
| 世界模型 | World Model | 学习物理世界运行规律的模型 |
